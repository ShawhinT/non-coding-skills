This is the innovation behind modern reasoning models, research agents, and coding agents.

## Example: RAG Process (Educational + Video Bridge)

RAG is a central technique for building AI systems that make an impact.

However, most implement it the wrong way...

The typical approach is to grab an off-the-shelf library (e.g. LangChain, LlamaIndex, etc.), dump all your files in a vector store, and connect it to an LLM.

While this can make for a quick and easy prototype, it will inevitably fail as usage scales.

Instead, here's how I approach building RAG systems.

Step 1: Scope the MVP
Define the 1 user, 1 use case, and 1 data source that drives most of the value you are trying to generate

Step 2: Create Golden Dataset
Generate realistic question-doc pairs you can use to evaluate your retrieval system

Step 3: Build v0 Retrieval System
Build an initial version of your retrieval system (this can be a fast and dirty)

Step 4: Build v0 RAG System
Connect the v0 retrieval system to an LLM and write a system prompt for your use case

Step 5: Run Experiments
Systematically evaluate your system using your golden dataset, RAG evals, and AI evals

In a recent YouTube video, I walk through these steps with a concrete example of building an answer engine based on my content.

👉 Check it out here: https://lnkd.in/gCgEvgNS

## Example: Claude Code Automation (Builder's Log + Video Bridge — Shaw's actual post)

Most people use Claude Code to write code.

I've been using it to automate my non-coding work :P

Here's how:
1) Do it once - complete the task with Claude in a conversation
2) Reflect - have Claude distill what worked into a reusable Skill
3) Test - run the Skill on a new example
4) Iterate - improve the Skill based on each run

Using this framework, I've automated:
- Cold outreach research
- Keynote presentations
- Writing Emails
- Managing my CRM
- Editing YouTube Videos (huge)

Each Skill gets better with every iteration (so Claude gets smarter every time I use it).

In a recent YouTube video, I walk through the full framework with concrete examples.
