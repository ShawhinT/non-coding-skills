## Example: 30 AI Buzzwords (Listicle Promo — value-first, P.S. links to free guide)

30 AI Buzzwords (Simply) Explained for Busy Leaders…

1. LLM = Software that can perform arbitrary tasks via English
2. Parameter = A number that controls what an LLM outputs given an input
3. Prompt = The request you pass to an LLM
4. Prompt Engineering = Crafting your prompt to optimize task performance
5. System Message = The prompt for an LLM app you are developing
6. Token = A unit of text that an LLM can understand
7. Inference = Using an LLM to generate text
8. Context Window = The maximum amount of text an LLM can process
9. Context Engineering = Filling the context window with the right info at the right time
10. Prompt Injection = When someone tricks an LLM into breaking its rules
11. Guardrails = Code-based rules you apply to LLM inputs and outputs
12. Hallucination = When LLMs make up things like facts and references
13. RAG (Retrieval-Augmented Generation) = Automatically giving LLMs context to complete a specific request
14. Chunking = Transforming source documents into text snippets
15. Embedding = A collection of numbers representing a text's meaning
16. Semantic Search = Search based on a query's meaning, not keywords
17. Vector Database = A collection of chunks + their embeddings
18. Hybrid Search = Combining keyword + semantic search for better results
19. Evals = Numbers that tell you if your AI system is good or not
20. Golden Dataset = Test cases for evaluating (or improving) your AI app
21. AI Agent = An LLM system that can use tools to perform actions
22. Agentic AI = An LLM system with some level of agency
23. MCP (Model Context Protocol) = A universal way to connect tools and context to LLMs
24. A2A (Agent to Agent) = A standard way to get AI agents to work together
25. Multi-agent System = Have multiple AI agents collaborate on a task
26. Fine-tuning = Adapting a model to a specific use case via additional training
27. SLM (Small Language Model) = An LLM with <10B parameters
28. Train-time compute = The cost of creating (i.e. training) an LLM
29. Test-time compute = The cost of using an LLM (after it's trained)
30. Reasoning Models = An LLM that can "think" before responding

What buzzword is missing here? Let me know in the comment and I'll explain :)
